Проанализируй весь мой проект и текущую систему логотипов на сайте.
В проекте появились новые SVG-файлы:

* `logo-text.svg`
* `logo-germes.svg`

Я хочу использовать их как основной логотип сайта вместо текущих логотипов.

Необходимо:

1. Найти ВСЕ места в проекте, где используются текущие логотипы:

   * header
   * mobile menu
   * footer
   * preloaders
   * favicon/manifest
   * Open Graph изображения или логотипы
   * schema.org organization logo
   * JSON-LD
   * SVG sprite/system
   * canvas/webgl/logo animations
   * CSS background-image
   * JS динамическая подгрузка логотипов
   * meta tags
   * PWA assets
   * sitemap/images если используются

2. Проанализировать:

   * размеры текущих логотипов
   * viewBox
   * aspect ratio
   * прозрачность
   * адаптивность
   * retina/display scaling
   * inline SVG или external SVG
   * влияние на layout
   * CLS (layout shift)
   * lazy loading
   * accessibility (`aria-label`, `title`, `role`)
   * SEO влияние

3. Проверить:

   * можно ли БЕЗОПАСНО заменить текущие логотипы на:

     * `logo-text.svg`
     * `logo-germes.svg`
   * не сломается ли:

     * верстка
     * responsive layout
     * header height
     * mobile navbar
     * sticky header
     * burger menu
     * footer grid
     * dark/light themes
     * hover states
     * animation logic
     * JS calculations
     * lazy loading
     * preload system

4. Определи:

   * какой SVG лучше использовать:

     * в header
     * в mobile menu
     * в footer
     * как favicon
     * для structured data
     * для social preview
   * нужны ли отдельные версии:

     * horizontal
     * icon-only
     * monochrome
     * dark mode
     * light mode

5. Проверь SVG-файлы на:

   * лишние группы
   * мусор из Figma/Illustrator
   * inline styles
   * transforms
   * clip-path проблемы
   * mask проблемы
   * stroke scaling
   * некорректные координаты
   * битые path
   * обрезанные элементы
   * совместимость браузеров
   * SVG optimization potential

6. Если необходимо:

   * автоматически оптимизируй SVG
   * исправь размеры
   * нормализуй viewBox
   * убери мусор
   * предложи production-ready версии

7. После анализа:

   * выведи полный отчет
   * перечисли все найденные зависимости
   * покажи какие файлы надо изменить
   * покажи какие компоненты используют логотип
   * покажи потенциальные риски
   * предложи лучший способ интеграции

8. Если замена возможна:

   * выполни полную миграцию логотипов по проекту
   * сохрани pixel-perfect отображение
   * не ломай существующую архитектуру
   * обнови все пути и импорты
   * сохрани SEO и accessibility

9. После изменений:

   * проверь проект на ошибки
   * проверь responsive
   * проверь все страницы
   * проверь Lighthouse-related проблемы
   * убедись, что логотипы отображаются корректно во всех состояниях сайта

Важно:

* не делать поверхностную замену
* сначала провести глубокий аудит
* учитывать production best practices
* не ломать существующую визуальную систему
* сохранить производительность сайта
* использовать clean SVG pipeline
* убедиться, что SVG корректно рендерятся в Chrome, Safari, Firefox и mobile browsers
