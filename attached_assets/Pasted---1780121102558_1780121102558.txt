Задача: создать секцию «Рекомендуемые объекты» на главной странице

Необходимо создать новую секцию каталога недвижимости на главной странице сайта агентства недвижимости «ГермесГарант».

Общие требования

Секция должна полностью соответствовать текущему дизайну сайта:

использовать существующую дизайн-систему;
использовать корпоративные цвета сайта;
использовать существующие переменные CSS;
соблюдать текущие отступы, радиусы, тени и типографику;
визуально выглядеть как органичная часть сайта, а не отдельный блок.
Источник данных

Все данные брать исключительно из файла:

objects.json

Отображать только объекты, у которых:

"recommended": true

На данный момент вывести любые 8 объектов из списка рекомендованных.

Заголовок секции

Пример структуры:

<section class="featured-properties">

Заголовок:

Рекомендуемые объекты недвижимости

Подзаголовок:

Актуальные предложения квартир, домов и участков в Лиде и Лидском районе.
Фильтрация объектов

В верхней части секции разместить вкладки-фильтры.

Список вкладок:

Все
Однокомнатные
Двухкомнатные
Трехкомнатные
Дома
Участки
Логика фильтрации
Все

Показываются все рекомендованные объекты.

Однокомнатные

Показывать объекты:

"type": "apartment"
"rooms": 1
Двухкомнатные

Показывать объекты:

"type": "apartment"
"rooms": 2
Трехкомнатные

Показывать объекты:

"type": "apartment"
"rooms": 3
Дома

Показывать:

"type": "house"
Участки

Показывать:

"type": "land"
Анимация вкладок

При переключении фильтра обязательно добавить плавную анимацию:

Активная вкладка
плавное изменение цвета;
плавное изменение фона;
transition 300ms.
Карточки

При смене фильтра:

opacity
transform: translateY()

Эффект:

старые карточки плавно исчезают;
новые карточки плавно появляются;
без резких перерисовок;
без мерцания.

Продолжительность:

0.3s – 0.4s
Сетка карточек

На десктопе:

4 карточки в ряд
2 ряда
Всего 8 карточек

Использовать CSS Grid.

Пример:

grid-template-columns: repeat(4, 1fr);
Планшет
2 карточки в ряд
Мобильные устройства
1 карточка в ряд
Карточка объекта

Карточки должны быть визуально идентичны карточкам на странице:

nedvizhimost-lida

Не создавать новый дизайн карточек.

Необходимо использовать тот же стиль:

изображение сверху;
скругления;
тень;
фирменные цвета;
структура контента;
бейджи;
типографика.

Карточки должны выглядеть абсолютно одинаково с каталогом недвижимости.

Содержимое карточки

Использовать данные из objects.json.

Отображать:

Изображение

Первое изображение объекта.

Название объекта

Например:

2-комнатная квартира в г. Лида на улице Тавлая
Краткое описание

Обрезать до 2–3 строк.

Цена

Пример:

114 523 BYN
Цена за м²

Если присутствует в данных.

Бейджи

Если есть соответствующие данные:

Рекомендуемый
Новинка
Снижение цены
Кнопка

Внизу карточки разместить кнопку:

Подробнее

Стиль кнопки использовать существующий на сайте.

Переход на страницу объекта

Кнопка должна вести на страницу объекта.

Использовать URL объекта из:

slug

или текущую логику генерации ссылок, которая уже применяется на странице каталога недвижимости.

Переход должен работать идентично странице:

nedvizhimost-lida
Производительность

После загрузки страницы:

загрузить objects.json один раз;
сохранить данные в памяти;
не выполнять повторные запросы при переключении вкладок;
фильтрацию выполнять на клиенте через JavaScript.
Дополнительные требования

Перед реализацией проанализировать существующие файлы проекта:

главную страницу;
карточки каталога недвижимости;
objects.json;
существующие CSS компоненты;
существующие JS модули.

Максимально переиспользовать существующие компоненты карточек и стили.

Не дублировать код карточек, если уже существует компонент рендера объектов.

Приоритет:

Полное соответствие стилю страницы nedvizhimost-lida.
Использование данных из objects.json.
Плавная фильтрация с анимацией.
Адаптивность.
Переиспользование существующей архитектуры проекта.
